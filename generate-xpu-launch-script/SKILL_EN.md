# XPU Training Launch Script Generator

This Skill guides the Agent to generate training launch scripts based on XPU YAML configuration files.

## Input Parameters

| Parameter | Required | Description | Example |
|-----------|----------|-------------|---------|
| `config_file` | Yes | XPU YAML configuration file path | `/data/configs/train_xpu.yaml` |
| `python_env_path` | Yes | Python virtual environment path | `/root/paddlejob/env/paddle` |
| `output_dir` | **Yes** | Training output directory, **must be specified by main agent** | `/root/paddlejob/tmp/output` |
| `model_name` | No | Model name, default read from YAML | `Qwen3-VL-30B` |
| `num_xpus` | No | Number of XPU devices, default 8 | `8` |
| `xpu_devices` | No | XPU device list, default `0,1,2,3,4,5,6,7` | `0,1,2,3,4,5,6,7` |
| `api_yaml_path` | No | API trace configuration path, auto-generated by default | `${OUTPUT_DIR}/api.yaml` |

## Execution Flow Overview

```
Input: config_file, python_env_path, [other optional parameters]
  │
  ▼
┌─────────────────────────────┐
│  Step 1: Read YAML Config   │
│  - Extract key information  │
│  - Determine default values │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Step 2: Prepare Template   │
│  - Collect all placeholder  │
│  - Calculate dynamic paths  │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Step 3: Fill Template      │
│  - Replace placeholders     │
│  - Generate script content  │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Step 4: Output Script      │
│  - Write file               │
│  - Add execute permission   │
│  - Generate usage guide     │
└─────────────────────────────┘
               │
               ▼
Output: Executable bash script path
```

## Step 1: Read YAML Configuration

**Execute**:
```bash
cat {config_file}
```

**Extract Information**:
- `model_name_or_path` → for `MODEL_NAME`
- ~~`output_dir` → no longer read from YAML, must use passed parameter~~
- Training parameters like `per_device_train_batch_size`, `gradient_accumulation_steps`

**Default Value Handling**:
| Parameter | Source | Default Value |
|-----------|--------|---------------|
| `MODEL_NAME` | Last segment of `model_name_or_path` | `unknown_model` |
| `OUTPUT_DIR` | **main agent passed `output_dir` parameter** | **None (required)** |

> **Important**: `output_dir` must be explicitly passed by the main agent. Reading from YAML configuration file is prohibited to ensure all training outputs are unified under the specified directory.

---

## Step 2: Prepare Template Variables

### Required Variables (User must provide)

| Template Variable | Value | Description |
|-------------------|-------|-------------|
| `{{PYTHON_ENV_PATH}}` | `{python_env_path}` | Python environment path |
| `{{CONFIG_FILE}}` | `{config_file}` | YAML configuration file path (absolute path) |
| `{{OUTPUT_DIR}}` | `{output_dir}` | **Training output directory (enforced by main agent)** |

### Auto-calculated Variables

| Template Variable | Calculation | Description |
|-------------------|-------------|-------------|
| `{{MODEL_NAME}}` | `{model_name}` or extract from YAML | Model name |
| `{{NUM_XPUS}}` | `{num_xpus}` or `8` | Number of XPU devices |
| `{{XPU_DEVICES}}` | `{xpu_devices}` or `0,1,2,3,4,5,6,7` | XPU device list |
| `{{GENERATED_TIME}}` | `$(date '+%Y-%m-%d %H:%M:%S')` | Generation time |
| `{{API_YAML_PATH}}` | `{api_yaml_path}` or `${OUTPUT_DIR}/api.yaml` | API trace configuration |
| `{{API_CONFIG_PATH}}` | `${OUTPUT_DIR}/api_config.txt` | API configuration file |
| `{{FLAGS_TRACE_API}}` | `${API_YAML_PATH},${API_CONFIG_PATH}` | API trace flags |
| `{{LD_LIBRARY_PATH}}` | `${PYTHON_ENV_PATH}/lib/python${VERSION}/site-packages/paddle/libs/` | Dynamic library path |

### Python Version Detection

**Execute**:
```bash
{python_env_path}/bin/python --version
```

**Extract version number** (e.g., `3.10`) for constructing `LD_LIBRARY_PATH`

---

## Step 3: Fill Template

### Template File Location

```
templates/xpu_train.sh.template
```

### Template Placeholder List

| Placeholder | Source | Example Value |
|-------------|--------|---------------|
| `{{GENERATED_TIME}}` | Auto-generated | `2025-01-15 10:30:00` |
| `{{MODEL_NAME}}` | User provided or YAML | `Qwen3-VL-30B` |
| `{{CONFIG_FILE}}` | User provided | `/data/configs/train_xpu.yaml` |
| `{{OUTPUT_DIR}}` | User provided or YAML | `./checkpoints/train_001` |
| `{{NUM_XPUS}}` | User provided or default | `8` |
| `{{XPU_DEVICES}}` | User provided or default | `0,1,2,3,4,5,6,7` |
| `{{PYTHON_ENV_PATH}}` | User provided | `/root/paddlejob/env/paddle` |
| `{{API_YAML_PATH}}` | User provided or generated | `./checkpoints/train_001/api.yaml` |
| `{{API_CONFIG_PATH}}` | Auto-calculated | `./checkpoints/train_001/api_config.txt` |
| `{{FLAGS_TRACE_API}}` | Auto-concatenated | `./checkpoints/train_001/api.yaml,./checkpoints/train_001/api_config.txt` |
| `{{LD_LIBRARY_PATH}}` | Dynamic construction | `${PYTHON_ENV_PATH}/lib/python3.10/site-packages/paddle/libs/` |

### Filling Steps

1. **Read template file content**
2. **Replace placeholders one by one**:
   - Replace `{{MODEL_NAME}}` with actual model name
   - Replace `{{PYTHON_ENV_PATH}}` with actual environment path
   - ...and so on
3. **Ensure `LD_LIBRARY_PATH` uses variable form**:
   ```bash
   export LD_LIBRARY_PATH=${PYTHON_ENV_PATH}/lib/python3.10/site-packages/paddle/libs/:${LD_LIBRARY_PATH}
   ```

## Step 4: Output Script

### Determine Output Path

- Default: `{config_file directory}/train_xpu.sh`
- Or user-specified path

### Write File

```bash
cat > {output_script_path} << 'EOF'
{Filled script content}
EOF
```

### Add Execute Permission

```bash
chmod +x {output_script_path}
```

### Generate API Configuration Files (Optional)

If `api_yaml_path` not provided, create default files:
```bash
mkdir -p {OUTPUT_DIR}
cat > {OUTPUT_DIR}/api.yaml << 'EOF'
apis: []
EOF

cat > {OUTPUT_DIR}/api_config.txt << 'EOF'
EOF
```

## Notes

1. **PYTHON_ENV_PATH Required**: This is the only parameter that must be manually specified, used to activate Paddle environment
2. **Use Absolute Paths**: Recommend using absolute paths to avoid working directory switching issues
3. **LD_LIBRARY_PATH Dynamic Generation**: Prohibited from hardcoding fixed paths, must be dynamically constructed based on `PYTHON_ENV_PATH`
4. **API Configuration Files**: If `api_yaml_path` not provided, default files will be automatically created under `OUTPUT_DIR`
5. **Execute Permission**: Automatically add `+x` execute permission after generating script
6. **Environment Validation**: Script will verify if `PYTHON_ENV_PATH/bin/activate` exists at startup
